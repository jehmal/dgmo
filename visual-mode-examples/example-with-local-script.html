DGMO Visual Mode - Local Script Example

Avoid CORS issues by downloading and serving the Stagewise toolbar locally

This method works even with file:// protocol by using a locally downloaded toolbar script.

Step 1: Download Stagewise Toolbar

First, download the toolbar files to your project:

# Create a vendor directory
mkdir -p vendor/stagewise

# Download the toolbar script
curl -o vendor/stagewise/toolbar.js \
  https://unpkg.com/@stagewise/toolbar@0.4.9/dist/index.js

# Download the toolbar styles
curl -o vendor/stagewise/toolbar.css \
  https://unpkg.com/@stagewise/toolbar@0.4.9/dist/style.css
Note: Make sure to download both the JavaScript and CSS files to the same directory.

Step 2: Update Your HTML

Reference the local files instead of the CDN:

<!-- Local Stagewise Toolbar -->
<script type="module" src="vendor/stagewise/toolbar.js"></script>
<link rel="stylesheet" href="vendor/stagewise/toolbar.css">

<!-- Add the toolbar element -->
<stagewise-toolbar framework="vanilla"></stagewise-toolbar>

Step 3: Start DGMO

dgmo run --visual "help me improve this page"

Alternative: Using DGMO's Built-in Server

You can also use DGMO's visual server directly:

# This starts both DGMO and a development server
dgmo visual serve --port 3000

# Then open http://localhost:3000 in your browser
