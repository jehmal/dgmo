This example demonstrates using DGMO Visual Mode with an HTTP server to avoid CORS issues.

Step 1: Start DGMO with Visual Mode

dgmo run --visual "help me improve this UI"

Step 2: Serve this file via HTTP

Choose one of these methods:

# Python (most common)
python -m http.server 8000

# Node.js
npx http-server -p 8000

# PHP
php -S localhost:8000

# Ruby
ruby -run -e httpd . -p 8000

Step 3: Open in Browser

Navigate to: http://localhost:8000/example-with-server.html

Step 4: Use Stagewise Toolbar

The toolbar should appear at the bottom of the page. Select any element and add comments!

Why HTTP Server?

When you open HTML files directly (file:// protocol), browsers enforce strict CORS policies that block external scripts from loading. This is a security feature to prevent malicious local files from accessing web resources.

By serving the file through an HTTP server, you get a proper origin (http://localhost:8000) which allows the Stagewise toolbar script to load correctly.
